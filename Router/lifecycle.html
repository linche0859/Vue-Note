# 生命週期

注意

App.vue 本身是所有程序的進入點，而且並沒有符合整個根目錄的任何元件或是執行方法。

所以，App.vue 自身是沒有辦法使用 Router 的 Hooks 方法的。

# 順序

  1. beforeCreate, created, beforeMount 順序 父元件 > 子元件。

    舉例：一父元件進入另一父元件的子元件，會先執行另一父元件的beforeCreate, created, beforeMount才是子元件的。

  2. mounted 順序 子元件 > 父元件。

    範例如上

  3. 同父元件，以下事件只會做 1 次：

    1. beforeEnter
    2. beforeRouteEnter
    3. beforeCreate
    4. created
    5. beforeMount
    6. mounted

  4. 子元件在完全離開時才會執行 beforeRouteLeave。

  5. 在路由當中「重新整理頁面」的順序不太一樣，以 /hello/kitty 列舉如下：

    1. 先執行父元件 Hello.vue 的 beforeEnter。

    2. 接著子元件 HelloKitty.vue 的 beforeEnter。

    3. 然後才是 App.vue 的系列動作：

      1. beforeCreate
      2. created
      3. beforeMount
      4. mounted

    4. 接著是 Hello.vue 的 beforeRouteEnter

    5. 接著子元件 HelloKitty.vue 的 beforeRouteEnter

    6. 全局的 beforeResolve

    7. 再來是 App.vue 的 beforeUpdate

    8. 後續是 Hello.vue 、 HelloKitty.vue 與其他元件系列動作 (父元件 > 子元件)

      1. beforeCreated
      2. created
      3. beforeMount

    9. 最後在依照路由順序 mounted，內部元件理論上會優先 (子元件 > 父元件)

    10. 最後的最後 App.vue 的 updated
